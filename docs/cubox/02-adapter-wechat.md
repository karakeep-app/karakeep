# WP-02 微信公众号适配器

> **依赖**：WP-01 适配器框架。
> **难度**：低。建议作为 WP-01 的"首发示例适配器"一起交付。

## 范围

URL 模式：`mp.weixin.qq.com/s/*` / `mp.weixin.qq.com/s?*`

抓取目标：
- 标题、作者（公众号名）、发布时间
- 清洗后的正文（保留段落、图片、视频占位）
- 封面图、图片列表

## 实现线索（仅指方向，不规定步骤）

- 公众号页面公开、HTML 结构稳定多年；优先 **HTTP GET + Cheerio**，无需 Playwright
- 关键 DOM：标题 `#activity-name`、作者 `#js_name`、时间 `#publish_time`、正文容器 `#js_content`
- OG 标签可作为冗余兜底
- **图片防盗链**：`mp.weixin.qq.com` 下载需带 `Referer` 头，否则 403。强烈建议下载后转存到 assetdb（与 WP-06 协同；MVP 阶段可先同步下载）
- 兜底走 Playwright（仅在出现风控页时启用）

## 验收

- 黄金集 5 条真实文章（覆盖纯文、图文、含视频、含外链卡片、长文）
- title/cover/正文/图集 100% 非空；正文段落数 ≥ 实际值的 90%
- P50 < 1s、P95 < 3s（不含图片下载）
- 防盗链处理验证：抓取的图片可在 Karakeep 阅读视图内正常显示

## 风险

- 偶尔触发"环境异常"验证页 → 走 Playwright 兜底
- 图片 CDN 偶尔变更（参数签名）→ 落地本地存储可规避

## 留给 agent 决定

- 是否同步下载图片，还是直接接 WP-06 异步管线
- 正文 HTML 是否做进一步清洗（去 JS、去 #js_xxxx 容器属性）
